Sana's Dev Hub: Dynamic E-Learning App

Your One-Stop Resource for Programming Mastery

Project Overview

Sana's Dev Hub is a comprehensive, native Android application meticulously designed to be a one-stop learning resource for students and aspiring developers. This platform provides structured courses on various computer science subjects, including Data Science, Mobile App Development, and Data Structures.

The app features a rich, intuitive user interface and delivers content through multiple innovative formats such as curated video tutorials, interactive quizzes, detailed text-based cheat sheets, and hands-on assignments. This multifaceted approach creates a holistic and engaging learning experience, empowering users to master complex programming concepts efficiently.

Key Features & Functionalities

Dynamic & Scalable Course Structure

  • The app is built with a scalable architecture, allowing the main dashboard to dynamically load subject categories from a list. This makes it effortless to add new courses like "C# Web Application" or "Artificial Intelligence" without altering the core app structure.

Rich & Diverse Content Modules

  • Curated Video Playlists: Direct links to YouTube playlists for in-depth video tutorials (via YouTube Intent).
  • Web-based Tutorials: Seamless integration of external learning resources like W3Schools tutorials (via custom Web Intent).
  • In-app Cheat Sheets: Access to scrollable, text-based "Cheat Sheets" for quick revisions of key concepts.
  • Practical Assignments: Structured assignment sections with separate tutorial and question components for hands-on practice.

Interactive Quiz System

  • A fully functional quiz engine that presents multiple-choice questions dynamically.
  • Progress Tracking: Monitors user progress with a ProgressBar and provides a detailed results screen with the final score, encouraging self-assessment.

Advanced Content Viewer (JSON-Powered)

  • Tutorial and cheat sheet content is dynamically parsed from local JSON files, structured to mimic Jupyter Notebooks. This allows for easy content updates and management without app redeployment.
  • The viewer intelligently renders: headings (H1, H2), paragraphs, and formatted code blocks with a convenient "Copy Code" button.

Resource & Solutions Hub

  • The assignment section features a sophisticated `TabLayout` and `ViewPager2` implementation, providing a dedicated tab for "Solutions & Resources."
  • Users can access supplementary materials like solution PDFs and CSV datasets (for practical assignments) directly via Google Drive links, enhancing the learning support.

Modern & Responsive UI/UX

  • The entire application adheres to Material Design principles, featuring custom handwritten fonts and a consistent, visually appealing color palette.
  • Utilizes MaterialCardView for clean, modern layouts, ensuring a smooth and intuitive user experience across all screens and device sizes.

Challenges & Key Learnings

Building Sana's Dev Hub presented critical challenges, fostering significant growth in mobile app development and problem-solving:

Tackling ANR (UI Freezing)

Problem: Parsing large JSON files on the main UI thread caused the app to freeze (ANR) or crash, especially on content-heavy screens like Assignments or Cheat Sheets.

Solution: Implemented background processing using java.util.concurrent.ExecutorService. All file I/O and JSON parsing operations were offloaded to a background thread, using runOnUiThread() to safely update the UI. A ProgressBar provided visual feedback during loading, completely resolved the ANR issue.

Efficient Content Display

Problem: The JSON content contained diverse data types (headings, paragraphs, code blocks). Displaying this dynamically within a ScrollView using static TextViews would be inefficient and hard to manage.

Solution: Designed a robust system using a RecyclerView with multiple view types. Created a ContentAdapter that could identify the content type (e.g., TYPE_HEADING, TYPE_PARAGRAPH, TYPE_CODE) and inflate the correct layout for each item. This made the content display dynamic, memory-efficient, and easily extensible.

UI Component Casting Issues

Problem: After updating a TextView to a MaterialCardView in the XML layout for a better UI, the app started crashing with a ClassCastException.

Solution: Debugged the RecyclerView.Adapter and identified that the ViewHolder was still trying to cast the view to a TextView. Corrected the variable type in the ViewHolder class from TextView to MaterialCardView, ensuring the Java code was in sync with the updated XML layout and preventing future casting errors.
