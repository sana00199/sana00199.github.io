CircleUp: Secure & Interactive Messaging

Your Private Circle for Encrypted & Engaging Communication

Project Overview

CircleUp is a robust, feature-rich Android messaging application developed as my Final Year Project. It stands out by meticulously focusing on user privacy and offering innovative real-time communication features, moving beyond the typical social media clutter. This project demonstrates strong proficiency in full-stack mobile application development, complex backend integration, advanced security implementation, and seamless real-time functionalities.

It was engineered to be a comprehensive communication platform, showcasing the ability to design, build, and deploy a secure and engaging mobile experience from concept to completion.

Addressing Real-World Challenges

The Privacy Dilemma

In an era of increasing data breaches and intrusive social platforms, users demand truly private and secure conversations without compromising usability.

Generic Communication Tools

Most messaging apps offer standard features, lacking the unique, interactive elements that can truly enrich digital dialogue.

Complex App Management

Maintaining user safety and app integrity requires robust administrative tools, which are often overlooked in smaller projects.

CircleUp's Solution

CircleUp was engineered to be a private communication hub, emphasizing End-to-End Encryption from the ground up. It pioneers unique interactive features like Shared Drawing and Collaborative Browsing, transforming passive messaging into engaging experiences. Furthermore, it incorporates a comprehensive Admin Dashboard, ensuring meticulous user and content management for a secure and trusted environment.

Core & Advanced Features

Security & Privacy Foundations

  • End-to-End Encryption (E2EE): Pioneered robust E2EE for one-to-one chats, ensuring messages are encrypted and decrypted strictly on user devices (AES for symmetric encryption, RSA for secure key exchange and management).
  • Passphrase-Protected Chats: Messages are secured by a user-defined passphrase/PIN, making them inaccessible without it, even if the device is compromised. This passphrase is required every login attempt to unlock chats.
  • Secure Account Recovery: Implemented a mechanism involving a unique recovery code for account retrieval.
  • Real-time Block Status Monitoring: Accounts disabled by admins are immediately logged out and prevented from re-accessing the app until re-enabled.

Interactive & Engaging Communication

  • Shared Drawing: A unique real-time collaborative whiteboard feature, allowing chat participants to draw and doodle together.
  • Invisible Ink: Messages appear hidden and are only revealed upon user interaction, adding a playful and private element.
  • Collaborative Web Browsing ("Let's Search Together"): Enables users to jointly browse a web page within the chat, sharing real-time search queries and web interactions.
  • Voice Notes: Seamless recording and sharing of voice messages, securely stored on Supabase.
  • Scheduled Messages: Ability to compose and schedule text or image messages for future delivery using WorkManager for reliability.
  • Temporary Chat Rooms: Create time-limited chat environments that automatically expire, ideal for short-term discussions.

Core Messaging & Management

  • One-to-One Messaging: Standard direct text and media exchange.
  • Group Chats: Create, manage, and participate in private group conversations.
  • Media Sharing: Send and receive images, videos, and documents securely.
  • Real-time Status Updates: Instant delivery confirmation, 'seen' receipts, and user online/offline status.
  • Drafts Management: Conveniently save and manage message drafts.

Comprehensive Admin Dashboard

A separate, secure web-based module providing powerful tools for application oversight:

  • User Management: View, manage, disable, and enable user accounts.
  • Reported Users Management: Review detailed reports from users, including reasons, reporter, and timestamp; delete reports.
  • Feedback System: View and respond to user feedback submitted through the app.
  • Global Broadcasts: Send messages to all application users.
  • Group Oversight: View all created groups.
  • Admin Profile & Security: Secure admin authentication and profile management.

Technology Rationale & Key Decisions

For CircleUp, conscious technology choices were made to optimize for cost-efficiency, security, and performance, moving beyond standard Firebase offerings when better alternatives aligned with project goals:

Cloud Storage: Firebase Storage vs. Supabase Storage

While Firebase offers integrated storage, its pricing model for heavy media usage can become costly. Supabase Storage was chosen as a more cost-effective and scalable solution for storing user-generated media like voice notes, images, and documents, offering robust file management with competitive pricing.

Push Notifications: Firebase Cloud Messaging (FCM) vs. OneSignal

Though FCM is Firebase's native solution, OneSignal was preferred for its extensive free tier features, user-friendly dashboard for campaign management, and advanced analytics. It provided a powerful and developer-friendly alternative for reliable push notifications without incurring immediate costs associated with high-volume FCM usage.

Scheduled Tasks: Firebase Cloud Functions vs. WorkManager

For handling scheduled messages (like sending messages at a future time), WorkManager was utilized over server-side solutions like Firebase Cloud Functions. WorkManager offers a robust, on-device solution for deferrable background tasks, significantly reducing server-side costs and simplifying deployment, as it doesn't require managing server infrastructure or cloud function invocations for each scheduled message.

Feedback Storage: Firebase Realtime Database vs. Firestore

While most real-time chat data resides in Firebase Realtime Database, Firestore was specifically chosen for storing and managing user feedback. Firestore's document-model excels in handling structured data, complex queries, and provides better scalability for administrative tasks like filtering and searching feedback, making it ideal for the Admin Dashboard's requirements.

Challenges & Key Learnings

Navigating the complexities of building a secure, real-time messaging platform presented invaluable challenges and insights:

Implementing E2EE

Ensuring correct AES & RSA key generation, secure storage, and real-time exchange was technically demanding. Debugging decryption failures due to key mismatches or data corruption was a significant learning curve.

Real-time Synchronization

Achieving seamless, low-latency updates for messaging, shared drawing, and browsing, while managing Firebase listeners efficiently, required meticulous state management and concurrency handling.

Offline & Data Consistency

Integrating Room Database for offline access and ensuring consistent data synchronization between local storage, Firebase, and UI was complex, especially with encrypted content.

Admin Security & Control

Developing a separate, secure admin panel with robust user moderation (e.g., real-time account blocking/enabling) required careful permission management and system design.
