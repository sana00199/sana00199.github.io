CircleUp: Next-Gen Social Messaging App (Final Year Project)

Project Overview:

Developed 'CircleUp,' a robust and feature-rich Android messaging application in Java as my Final Year Project. This application goes beyond basic messaging to provide a secure and comprehensive communication platform, showcasing a strong command over various Android development concepts, backend integration, security implementation, and real-time features. CircleUp demonstrates the ability to build complex, full-stack mobile applications.

Key Technical Stack:

Implemented the application using a diverse set of modern technologies:

  • Language: Java
  • IDE: Android Studio
  • UI/Layout: XML Layouts
  • Backend (Authentication & Realtime Database): Firebase Realtime Database for real-time data synchronization (messages, user status, etc.) and Firebase Authentication for user registration and login.
  • Backend (Firestore for Feedback): Utilized Firestore for managing user feedback data efficiently.
  • Local Persistence: Room Database for robust offline data storage and caching.
  • Media Storage: Supabase for storing media files, including videos and voice notes.
  • Push Notifications: Integrated OneSignal SDK for sending and managing push notifications (e.g., new message alerts, friend requests).
  • Background Tasks: Implemented scheduled messages using WorkManager for reliable background execution.
  • Version Control: Git and GitHub for collaborative development and code management.
  • APIs/Concepts: RESTful APIs (implied by Firebase/Supabase interaction), JSON data handling.

Core Messaging & Communication Features:

  • One-to-One Chats: Direct, private messaging between users.
  • Group Chats: Create and participate in group conversations.
  • Temporary Chat Rooms: Feature for time-limited conversations (including voice note implementation stored on Supabase).
  • Scheduled Messages: Ability to schedule text or image messages to be sent at a later time using WorkManager.
  • Real-time Updates: Instant message delivery and status updates using Firebase Realtime Database.

Advanced & Unique Features:

  • End-to-End Encryption (E2EE): Implemented E2EE for one-to-one chats to ensure message privacy. Messages are encrypted/decrypted locally on devices. (Likely involving concepts like AES for symmetric encryption and RSA for key exchange/management).
  • Secure Access with Passphrase: Chats are protected by a user-defined security passphrase/PIN. Messages cannot be decrypted without this passphrase, even if the device is compromised, demonstrating a strong focus on user data security.
  • Account Recovery: Implemented an account recovery mechanism, including generating and using a recovery code (though resetting passphrase results in loss of old encrypted messages, highlighting the importance of the original passphrase).
  • Shared Drawing Feature: A collaborative feature allowing users in a chat to draw together in real-time.
  • Invisible Ink Feature: Messages initially appear hidden and are revealed by user interaction.
  • Shared Web View Feature: Allows both users in a chat to view and interact with a web page simultaneously, enabling collaborative browsing or content sharing (like your "Let's get started together" feature).

Admin Panel (Separate Module):

Developed a dedicated Admin Dashboard module to manage the application and users:

  • View Notifications (alerts, reports).
  • Broadcast Messages to all users.
  • View and manage all registered users.
  • View and respond to user feedback (data stored in Firestore).
  • View and manage reported users (e.g., disable accounts).
  • View and manage disabled users (e.g., enable accounts).
  • View all administrator accounts.
  • View all created groups.
  • Admin Profile Management.
  • Secure Admin Authentication.
